# 萌宠日记项目说明 (MengChongRiJi Project)

本 skill 以“功能清单”为主线，帮助 Agent 快速理解当前 uniapp 项目的功能、页面与数据约定。

## 📅 项目定位与核心需求

- **产品名**：萌宠日记
- **定位**：纯粹的宠物行为记录与成长助手，旨在去除所有商业化及社交干扰。
- **核心场景**：
  1. **多宠物管理**：用户可创建多个宠物档案，记录各自独立的数据。
  2. **全面行为记录**：支持包括喂食、饮水、遛弯、便便、嘘嘘、心情、睡眠、医疗、培训、玩耍、零食、洗澡、疫苗、驱虫、体重等在内的所有常见宠物行为。
  3. **备注与多媒体**：支持详细的文字备注及多图上传。
  4. **历史与统计**：提供完整的历史记录列表及月度行为数据看板。
  5. **智能提醒**：在记录时支持设置下一次提醒（如疫苗、驱虫），并在首页直观展示。
  6. **云端同步**：通过 Node.js + SQLite 服务器同步所有档案与记录。

## 📂 技术栈与结构

- **前端**：uni-app (Vue 3 Options API)，主色调为温馨暖橙色。
- **后端**：Node.js + Express + SQLite，表结构包括 `pets`, `records`, `reminders`, `users`。
- **存储**：默认使用本地持久化，配置 `utils/config.js` 后支持 API 云端同步。

## 📋 功能清单

### 一、宠物管理
- **宠物档案**：支持添加/编辑/删除宠物（名字、品种、性别、生日、体重、头像）。
- **多宠切换**：首页展示所有宠物，支持查看各自的最新记录。

### 二、全方位行为记录
- **扩展行为类型**：共有 16 种行为类型，支持自定义备注内容。
- **智能 Placeholder**：根据不同行为类型，提供精准的备注引导。
- **图片记录**：支持单条记录上传最多 3 张图片。
- **关联提醒**：记录的同时可一键创建下一次相关的待办提醒。

### 三、历史与统计
- **历史列表**：支持按宠物、按记录类型进行多重筛选。
- **月度看板**：自动统计本月各类行为的频度，直观展示宠物的活跃情况。

### 四、提醒中心
- **待办展示**：首页滑动展示即将到来的提醒事项。
- **状态流转**：支持将提醒标记为已完成。

## 🛣️ 路由与页面映射

| 路径 | 说明 |
|------|------|
| `pages/index/index` | 首页（包含待提醒事项、快捷记录入口、我的宠物列表） |
| `pages/pet/add` | 添加/编辑宠物档案 |
| `pages/pet/detail` | 宠物详情页 |
| `pages/record/add` | 添加行为记录（核心功能页，支持多宠选择、备注、多图、提醒） |
| `pages/record/list` | 历史记录列表（支持筛选） |
| `pages/record/statistics` | 行为数据统计报表 |
| `pages/reminder/index` | 提醒中心管理 |

## 📦 数据约定 (Local Storage)

- **`pets`** (Array): 存储宠物对象。
- **`records`** (Array): 存储行为记录对象。
- **`reminders`** (Array): 存储待办事项对象。

## ⚠️ 开发注意事项

1. **去商业化**：严禁在项目中加入任何下单、支付、上门服务或具有商业导向的功能。
2. **多宠逻辑**：所有记录必须关联 `pet_id`。
3. **性能优化**：由于服务器资源有限，图片上传应进行适度压缩，且记录列表需支持合理的分页/筛选。
